飞书 飞书多维表格设置
▶
根据你的飞书账号类型选择，国内用户选飞书，海外用户选 Lark
在飞书开放平台创建应用获取
从多维表格 URL 中获取：feishu.cn/base/bascnxxx?table=...
从 URL 中获取：?table=tblxxx&view=...
不勾选则只保存文本摘要，无需 drive 权限

1. 必须的权限（均为免审）：
bitable:app（多维表格读写）

2. 上传附件的权限（免审）：
drive:file:upload（上传文件到云空间）

3. 多维表格字段：
标题（文本）、链接（超链接）、作者（文本）、保存时间（日期）、评论数（数字）
如勾选上传附件：附件（附件）；否则：正文（文本）

Notion Notion Database 设置 新
▶
在 Notion Integrations 创建获取 · 详细教程
从 Database 链接中复制，格式为 32 位字符（可含连字符）
属性名需与 Notion Database 中的列名完全匹配（区分大小写）

配置步骤：
1. 创建 Internal Integration 并复制 Token
2. 创建 Database 并添加以下属性列（名称和类型必须匹配）：
   • 标题（Title 类型）- 必填
   • 链接（URL 类型）- 必填
   • 作者（Rich Text 类型）
   • 分类（Rich Text 或 Select 类型）
   • 保存日期（Date 类型）
   • 评论数（Number 类型）
3. 在 Database 页面点击「...」→「Connections」→ 添加你的 Integration
4. 从 Database 链接中复制 ID 并填入上方

注意：启用后图片会以 Base64 格式嵌入 Markdown 文件中。优点是单文件完整保存，缺点是文件体积会显著增大。

评论设置
▶
条（0-10000）

⚠️ 评论较多时可能需要较长时间加载

评论获取说明：
• 评论数 ≤30 条：从当前页面提取（快速）
• 评论数 >30 条或勾选"保存全部"：通过API获取（完整，解决懒加载问题）
• 超过500条评论时会显示加载进度

使用方法：
- 单击链接按钮 → 保存到 Obsidian/飞书
- 双击链接按钮 → 复制链接
- Ctrl+Shift+S（Mac: ⌘+Shift+S）→ 快捷键保存
