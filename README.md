# LinuxDo to Obsidian V3.5.5

**劫持LinuxDo书签按钮**，一键保存帖子+评论到 **Obsidian** 或 **飞书多维表格**，支持双保存。

## V3.5 版本新功能

| 特性 | 说明 |
|-----|------|
| 飞书多维表格 | **同步保存到飞书，支持 MD 附件上传** |
| 飞书/Lark双版本 | **支持国内版(feishu.cn)和国际版(larksuite.com)** |
| 评论书签支持 | **点击评论书签保存主帖+该评论** |
| 单击/双击分离 | **单击保存，双击触发 L 站原生收藏** |
| 插件开关 | **可关闭插件，恢复书签原功能** |
| 文件名优化 | **只用标题命名，不再带帖子 ID** |

---

## 功能特性

- **劫持书签按钮**：单击保存到 Obsidian/飞书，双击添加 L 站收藏
- **评论书签支持**：点击评论区的书签，保存主帖+该评论（文件名带楼层号）
- **飞书同步**：自动保存到多维表格，支持 MD 文件附件
- **飞书双版本**：支持国内版(feishu.cn)和国际版(larksuite.com)
- **保存评论**：可配置保存 1-3000 条评论
- **评论折叠**：使用 `<details>` 标签，展开查看
- **完全保留样式**：文字颜色、代码高亮、表格样式完整保留
- **中文元数据**：frontmatter 使用中文字段
- **快捷键备用**：支持 `Ctrl+Shift+S` (Mac: `Cmd+Shift+S`)
- **插件开关**：可随时禁用，恢复书签原功能

## 安装方法

1. 下载本插件的所有文件到本地文件夹
2. 打开 Chrome 浏览器，访问 `chrome://extensions/`
3. 开启右上角的「开发者模式」
4. 点击「加载已解压的扩展程序」
5. 选择插件文件夹 `linuxdo-to-obsidian`

## 使用方法

### 核心操作

| 操作 | 效果 |
|-----|------|
| **单击** 主帖书签 | 保存主帖到 Obsidian/飞书 |
| **单击** 评论书签 | 保存主帖 + 该评论（文件名带楼层号） |
| **双击** 任意书签 | 添加到 L 站收藏（必须是同一个按钮） |
| **Ctrl+Shift+S** | 快捷键保存主帖 |

### 使用流程

1. 访问任意 LinuxDo 帖子页面
2. 点击评论区上方右下角的"三个点"菜单
3. 单击"书签"选项 → 保存到 Obsidian/飞书
4. 双击"书签"选项 → 添加 L 站收藏

## 配置选项

点击 Chrome 扩展图标 → 右键 → 「选项」

### 插件状态

| 配置项 | 说明 |
|-------|------|
| 启用插件 | 关闭后书签按钮恢复原功能（需刷新页面） |

### 保存目标

| 配置项 | 说明 |
|-------|------|
| 保存到 Obsidian | 启用 Obsidian 保存 |
| 保存到飞书多维表格 | 启用飞书同步 |

### Obsidian 设置

| 配置项 | 说明 |
|-------|------|
| Vault 名称 | 留空使用当前打开的 vault（推荐） |
| 保存文件夹 | 保存到 vault 中的哪个文件夹 |
| 使用 Advanced URI | 支持大内容保存（推荐开启） |

### 飞书设置

| 配置项 | 说明 |
|-------|------|
| API 版本 | 选择飞书国内版或 Lark 国际版 |
| App ID | 飞书开放平台应用 ID |
| App Secret | 飞书开放平台应用密钥 |
| app_token | 多维表格 token（URL 中 `/base/` 后面的字符串） |
| table_id | 数据表 ID（URL 中 `?table=` 后面的字符串） |
| 上传 MD 附件 | 将完整内容作为 MD 文件附件上传 |

### 内容设置

| 配置项 | 说明 |
|-------|------|
| 添加元数据 | 是否添加中文 frontmatter |
| 保留图片链接 | 是否保留帖子中的图片 |

### 评论设置

| 配置项 | 说明 |
|-------|------|
| 保存评论区 | 是否保存评论（默认关闭） |
| 评论数量 | 1-3000 条，默认 100 |
| 折叠评论 | 使用 `<details>` 标签折叠 |

## 飞书配置教程

### 1. 创建飞书应用

1. 访问 [飞书开放平台](https://open.feishu.cn/)
2. 创建企业自建应用
3. 获取 App ID 和 App Secret

### 2. 配置权限（均为免审权限）

| 权限 | 说明 |
|-----|------|
| `bitable:app` | 多维表格读写（必须） |
| `drive:file:upload` | 上传文件（上传附件时需要） |

### 3. 获取表格信息

从多维表格 URL 中提取：

```
https://飞书域名/base/CVchbNPTraNcq1s3RqPcTJFJnDU?table=tblUP8jpxkIGiSWU&view=...
                      └─────────────────────────────┘      └───────────────┘
                              app_token                        table_id
```

### 4. 创建多维表格字段

| 字段名 | 类型 | 说明 |
|-------|------|------|
| 标题 | 文本 | 帖子标题 |
| 链接 | 超链接 | 原帖 URL |
| 作者 | 文本 | 发帖人 |
| 保存时间 | 日期 | 时间戳 |
| 评论数 | 数字 | 评论条数 |
| 附件 | 附件 | MD 文件（勾选上传附件时） |
| 正文 | 文本 | 内容摘要（不勾选上传附件时） |

## 常见问题

### Q: 评论没有全部保存？

**A:** 插件只能提取当前页面已加载的评论。如果帖子有很多评论，需要先滚动页面加载更多评论，再点击保存。

### Q: 点击书签后没反应？

**A:** 请检查：
1. Obsidian 是否已运行
2. 浏览器是否允许 `obsidian://` 协议
3. 按 F12 查看控制台错误

### Q: 内容过长保存失败？

**A:** 请在设置中启用"使用 Advanced URI 插件"，需要先在 Obsidian 中安装 Advanced URI 插件。

### Q: 飞书保存失败？

**A:** 请检查：
1. App ID 和 App Secret 是否正确
2. 是否已添加 `bitable:app` 权限
3. 应用是否已发布
4. 多维表格是否已添加应用为协作者

### Q: 如何恢复书签原功能？

**A:** 在设置中关闭"启用插件"开关，然后刷新页面。
